# MiniMax Token Plan Agent 需求与实现文档

## 1. 一句话目标

做一个“多模态 AI 客户端壳子”，用户填入 API Key 后可直接使用文本、语音、视频、图片、音乐能力。

## 2. 已实现范围

- 文本对话：多会话、会话本地持久化
- 语音合成：异步任务提交 + 轮询 + 播放
- 视频生成：异步任务提交 + 轮询 + 播放
- 图片生成：同步返回 base64 图像 + 任务记录
- 音乐生成：任务提交 + 结果播放
- 设置中心：本地保存 API Key
- 配置中心：支持默认配置与 `minimax.config.json` 覆盖

## 3. 配置化策略

### 3.1 目标

- 支持“默认值可用”
- 支持“按项目文件集中配置”
- 避免每个页面写死模型名和基础 URL

### 3.2 实现

- 默认配置：`src/config/appConfig.ts`
- 覆盖配置：`minimax.config.json`
- 合并方式：默认配置 + 用户配置深合并

### 3.3 可配置项

- `apiBaseUrl`
- `models.chatDefault` / `models.chatOptions`
- `models.voiceModel` / `models.voiceDefault` / `models.voiceOptions`
- `models.videoDefault` / `models.videoOptions`
- `models.imageDefault` / `models.imageOptions`
- `models.musicDefault` / `models.musicOptions`
- `audio.voice` / `audio.music`

## 4. 安全与隐私

- API Key 存储在浏览器本地持久化存储中
- 代码库中不提交真实密钥
- `.gitignore` 忽略常见环境文件、编辑器缓存、日志文件
- 发布前需执行敏感信息扫描，避免 key/token/certificate 泄露

## 5. API 对接清单

### 文本

- `/text/chatcompletion_v2`

### 语音

- `/t2a_async_v2`
- `/query/t2a_async_query_v2`
- `/files/retrieve`

### 视频

- `/video_generation`
- `/query/video_generation`
- `/files/retrieve`

### 图片

- `/image_generation`

### 音乐

- `/music_generation`

## 6. 工程约束

- 技术栈：Next.js + React + TypeScript + Zustand
- 代码风格：ESLint 约束，TypeScript 严格模式
- 验证命令：
  - `npm run lint`
  - `npm run typecheck`
  - `npm run build`

## 7. 后续迭代建议

- 增加后端代理层，实现服务端密钥托管
- 增加任务重试与取消能力
- 增加请求速率限制与错误分类提示
- 增加 `.env.example` 与部署参数说明
