# MiniMax Token Plan Agent：产品需求与架构文档

## 1. 文档目标

本文档用于定义项目的功能边界、架构方案、非功能约束与演进方向，作为开源协作与版本迭代的统一参考。

## 2. 项目定位

MiniMax Token Plan Agent 是一个多模态 AI Web 客户端模板，面向需要快速接入 MiniMax 能力的开发团队与开源贡献者，重点提供：

- 可直接运行的前端工作台
- 可打包为 Windows/macOS 桌面应用
- 可打包为 Android/iOS 原生壳应用
- 统一的 API 请求与异步任务处理模式
- 可覆盖的模型与参数配置机制
- 面向扩展的工程结构与状态管理策略

## 3. 功能范围

### 3.1 已实现能力

- 文本对话
  - 多会话管理
  - 会话列表收放
  - 提示词与 MCP 栏目支持展开与收缩
  - 消息本地持久化
  - 对话上下文窗口控制
  - 会话头部摘要、消息时间展示与单条消息复制
  - 输入框自适应高度，支持 Enter 发送 / Shift + Enter 换行提示
  - 输入区与底部状态栏做安全间距处理，避免键盘提示文案被遮挡
- 语音合成
  - 异步任务创建
  - 支持语速、音量、语调高级控制，对齐官方 skills
  - 统一轮询查询与结果下载
  - 在线播放
- 视频生成
  - 异步任务创建（支持文生视频、图生视频、首尾帧、主体参考视频四种模式）
  - 统一轮询查询与结果预览
  - 默认参数为 6 秒、768p
- 图片生成
  - 同步生成返回
  - 结果展示与下载
- 音乐生成
  - 纯音乐（无歌词模式）与歌词模式，与官方 skills 参数对齐
  - 支持 `music-2.5` 与 `music-2.6` 模型，默认使用 `music-2.6`
  - 音频结果播放
- 设置中心
  - 左侧菜单分组（系统设置 / 提示词管理 / Skills 管理 / MCP 管理）
  - 设置弹窗放大，适配更完整的配置管理场景
  - 增加工作台状态摘要卡片，集中展示 Prompt / 仓库 / 已应用 Skill 数量
  - API Key 默认不持久化，可选“记住密钥”后落盘
  - 便携版（Portable）可正确继承“记住密钥”配置，避免重复输入
  - 主题切换（浅色 / 深色）
  - 提示词管理（按 chat / voice / video / image / music 维度维护，支持主题与详细提示词）
  - 提示词新增/编辑支持长文本多行输入，适配复杂模板
  - 提示词列表与快捷预览支持 Markdown 渲染，并提供滚动查看长内容
  - 提示词列表支持搜索、编辑、删除与按模块统计
  - 桌面端数据持久化：使用 `electron-store` 将设置、提示词、技能库等核心状态落地到系统应用数据文件夹，避免被浏览器清空
  - API 请求层统一支持 429 频率限制指数退避重试
  - Skills 管理（添加仓库、添加 Skill、应用与取消应用，支持直接从 GitHub 仓库地址快速解析并导入）
  - Skills 区域支持按仓库过滤，并展示仓库数 / Skill 总数 / 已应用数
  - MCP 管理（快速添加/删除、服务启用开关、JSON 导入）
  - MCP JSON 导入支持 `mcpServers.<name>.command/args/env` 结构
  - “清空密钥”操作仅在系统设置视图中展示
- 文本对话 MCP 增强
  - 对话输入区支持 MCP 总开关与多服务选择
  - 启用后在请求中注入已选 MCP 服务上下文（兼容 endpoint 或 command+args 访问方式）
- 底部状态栏
  - 仅展示 MiniMax-M* 模型 Usage 与倒计时，优先按 `models.tokenPlanStatusModels` 配置项筛选
  - 详细面板完整展示 `/coding_plan/remains` 返回字段，支持纵向滚动与手动关闭
  - 支持手动实时刷新 + 1 分钟自动刷新余额，秒级倒计时更新
- 各功能页提示词快捷入口
  - 聊天、语音、视频、图片、音乐页支持按主题筛选并快速插入详细提示词
  - 支持显式“替换输入”与“追加插入”，避免误覆盖当前编辑内容
  - 支持展示当前模块提示词数量、筛选结果数量与选中提示词预览
  - 支持将当前输入一键收藏到对应能力域
- 全局导航
  - 左侧“MiniMax多模态工作台”导航支持收放
  - 去除顶部重复标题，仅保留放大后的工作台标识
  - 默认宽度进一步收窄，提升主工作区可视面积
  - 左上角工作台标识左移并增大，左侧菜单文字适度缩小以强化层级对比
  - 设置入口降低层级并抬高位置，避免与底部状态栏详细面板重叠
- 长任务等待优化
  - 统一请求默认超时调整为 120 秒
  - 异步任务轮询间隔提高，降低接口压力
  - 短暂查询异常不再立即标记任务失败
- 跨平台交付
  - Windows 与 macOS 桌面端打包（Electron + electron-builder，含 Windows Portable 绿色版）
  - 桌面端图标按平台独立配置：Windows 使用 `.ico`，macOS 使用原生图标资源，避免跨平台打包校验失败
  - Android 与 iOS 移动端壳打包（Capacitor）
- CLI 能力
  - 提供 `mmx` 终端命令入口（`bin/mmx`）
  - 支持 `auth login/status/logout` 与 `config show/set/export-schema`
  - 支持 `quota`、`text chat`、`image generate`、`video generate/task get/download`
  - 支持 `speech synthesize/voices`、`music generate`、`search`、`vision`
  - 支持 `--out`、`--download` 的文件落盘能力
- 企业级视觉与交互体系
  - 引入 Airtable 设计规范（深蓝字色、Airtable Blue 交互色、大圆角组件）
  - 增强的多层柔和阴影与正向字间距（Tracking）排版
  - 更直观的深浅色（Dark Mode）切换 Switch 控件

### 3.2 非目标

- 不提供后端密钥托管能力（当前版本）
- 不实现企业级用户体系、权限系统与计费网关
- 不覆盖 MiniMax 全量 API 能力

## 4. 系统架构

### 4.1 分层设计

- 展示层：`src/app` 与 `src/components`
  - UI 组件库：shadcn/ui（基于 Radix + Tailwind 的可组合组件）
- 状态层：`src/store`（Zustand）
  - `usePromptStore.ts`：提示词主题/详细内容管理，支持去重新增与就地编辑
  - `useSkillsStore.ts`：Skills 仓库、技能与应用状态管理
  - `useSettingsStore.ts`：系统设置与 MCP 服务配置管理（含 JSON 导入落盘）
- 能力层：`src/lib`
  - `apiClient.ts`：统一请求封装（超时、错误归一）
  - `taskPolling.ts`：统一异步轮询抽象
- CLI 层：`bin/mmx`
  - 统一参数解析、命令分发、配置读写与异步任务轮询
- 配置层：`src/config/appConfig.ts` + `minimax.config.json`
- 跨端打包层
  - `electron/main.mjs`：桌面容器启动与静态资源托管
  - `capacitor.config.ts`：移动端原生壳配置与资源目录声明
- 额度观测层
  - `src/lib/tokenPlan.ts`：余额查询、Usage 计算、进度条与颜色规则
  - `src/components/TokenPlanStatusBar.tsx`：全局底栏展示与自动刷新

### 4.2 关键设计决策

- API 请求统一封装，减少页面级重复代码
- 轮询逻辑抽象复用，避免多页面实现漂移
- 任务状态集中管理，并限制本地持久化体积
- 配置归一化处理，防止非法配置直接击穿业务页面
- 前端统一静态导出（`out/`）作为桌面与移动壳共享输入
- 通过统一设置中心承载系统参数、提示词与 Skills，降低多入口带来的状态分散问题

## 5. 配置策略

### 5.1 来源与优先级

1. 默认配置：`src/config/appConfig.ts`
2. 覆盖配置：`minimax.config.json`

最终通过合并与归一化输出运行时配置。

### 5.2 可配置项

- `apiBaseUrl`
- `models.chatDefault` / `models.chatOptions`
- `models.voiceModel` / `models.voiceDefault` / `models.voiceOptions`
- `models.videoDefault` / `models.videoOptions`
- `models.imageDefault` / `models.imageOptions`
- `models.musicDefault` / `models.musicOptions`
- `audio.voice` / `audio.music`

## 6. 安全与隐私

### 6.1 当前策略

- API Key 默认存储于会话级存储（关闭浏览器失效）
- 支持显式开启“记住密钥”持久化（用户可控）
- 关闭“记住密钥”时，持久化存储中不保留密钥内容
- 仓库不提交真实密钥、令牌、证书

### 6.2 生产建议

- 增加 BFF/代理层托管上游密钥
- 对接口调用增加限流、重试与审计日志
- 对输入输出增加安全过滤与风控策略

## 7. API 对接清单

- 文本：`/text/chatcompletion_v2`
- 语音：`/t2a_async_v2`、`/query/t2a_async_query_v2`、`/files/retrieve`
- 视频：`/video_generation`、`/query/video_generation`、`/files/retrieve`
- 图片：`/image_generation`
- 音乐：`/music_generation`
- 余额：`GET https://www.minimaxi.com/v1/api/openplatform/coding_plan/remains`
- 搜索：`/coding_plan/search`
- 视觉：`/coding_plan/vlm`

## 8. 质量要求

### 8.1 开发规范

- TypeScript strict 模式
- ESLint 检查通过
- 变更应保持模块边界清晰

### 8.2 验证基线

- `npm run lint`
- `npm run typecheck`
- `npm run build`

### 8.3 打包验证基线

- 桌面端本地运行：`npm run desktop:start`
- Windows/macOS 安装包构建：`npm run desktop:dist`
- Windows 安装包/绿色版构建：`npm run desktop:dist:win`
- macOS 安装包构建：`npm run desktop:dist:mac`
- 移动端资源同步：`npm run mobile:sync`
- Android 运行：`npm run mobile:android`
- iOS 运行：`npm run mobile:ios`
- Android 打包（Debug APK）：`npm run mobile:build:android`
- iOS 原生工程同步：`npm run mobile:build:ios`

### 8.4 CI 发布基线

- 工作流文件：`.github/workflows/release.yml`
- 触发条件：推送 `v*` Tag
- 发布产物：
  - Windows NSIS 安装包（`.exe`）
  - Windows Portable 绿色版（`.exe`，可直接运行）
  - macOS DMG（`.dmg`）
  - Android Debug APK（`.apk`）
  - iOS Simulator App 压缩包（`App-simulator.zip`，非签名 IPA）
- 发布动作：自动创建/更新 GitHub Release 并上传产物
- 版本策略：桌面端构建前将 `package.json` 版本同步为当前 Tag（如 `v1.0.3` → `1.0.3`），保证发布文件名版本一致
- 手动触发脚本：`publish-release.bat`（输入 Tag 后自动推送并触发发布流水线）
